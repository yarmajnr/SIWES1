Basics of Web Development

In this session, we focused on understanding the basics of HTML, the language used to create the structure of web pages.

Introduction to HTML

HTML (Hypertext Markup Language) is the standard markup language for creating web pages. It provides the structure for web content, allowing us to define elements such as headings, paragraphs, lists, and more.

Structure of an HTML Document

We discussed the basic structure of an HTML document, which includes the following elements:

  • <!DOCTYPE html>: Declares the document type and version of HTML.
  • <html>: The root element of an HTML page.
  • <head>: Contains meta-information about the document, such as the title.
  • <body>: Contains the content of the HTML document, such as text, images, and links.

Basic HTML Elements

We covered several basic HTML elements, including:

  • Headings (<h1> to <h6>): Used to define headings, with <h1> being the highest level.
  • Paragraphs (<p>): Used to define blocks of text.
  • Lists: Ordered (<ol>) and unordered (<ul>) lists to organize items.
  • Tables (<table>): Used to display data in a tabular format, with rows and columns.

Assignment

We were given an assignment to create a simple web page that includes the following elements:

  • A heading using the <h1> tag
  • A paragraph of text using the <p> tag
  • A hyperlink using the <a> tag
  • An image using the <img> tag

This assignment is designed to reinforce our understanding of HTML and help us practice creating basic web pages.

Introduction to CSS

This week, we explored the basics of CSS (Cascading Style Sheets) to understand how to style and layout web pages effectively.

Understanding CSS Basics

CSS is used to control the presentation of web pages, allowing us to apply styles such as colors, fonts, and spacing. CSS works by associating style rules with HTML elements, and it can be included in HTML documents in several ways:

  • Inline CSS: Adding styles directly to an HTML element using the style attribute.
  • Internal CSS: Placing CSS rules within a <style> tag in the <head> section of an HTML document.
  • External CSS: Linking to an external stylesheet using the <link> tag.

CSS Selectors and Properties

We learned about different types of CSS selectors, which are used to target HTML elements, and how to apply styles using various CSS properties. Some key selectors and properties include:

  • Selectors: Element, class, and ID selectors to target specific HTML elements.
  • Properties: color, background-color, font-size, margin, padding, and more.

Week 5 - Topic 1

Launching a Website on GitHub Pages

Setting Up GitHub Pages

To set up GitHub Pages for your repository, follow these steps:

  1. Navigate to your GitHub repository on the GitHub website.
  2. Click on the "Settings" tab.
  3. Scroll down to the "GitHub Pages" section.
  4. Select the branch you want to use for GitHub Pages (usually `main` or `master`).
  5. Choose the folder (e.g., `/root` or `/docs`) to publish.
  6. Click "Save" to enable GitHub Pages.

Uploading Files

To upload files to your GitHub repository, follow these steps:

  1. Navigate to your GitHub repository.
  2. Click on the "Add file" button.
  3. Select "Upload files" from the dropdown menu.
  4. Drag and drop your files or click "Choose your files" to select files from your computer.
  5. Click "Commit changes" to upload the files.

Configuring GitHub Pages

Once GitHub Pages is set up, you can configure it as follows:

  1. Ensure your HTML files are located in the root directory or the specified folder.
  2. Verify the URL for your GitHub Pages site: https://yourusername.github.io/webrepository
  3. Customize the settings if needed, such as theme or custom domain.

Committing Changes

To commit changes to your GitHub Pages site, follow these steps:

  1. Make changes to your files locally or directly on GitHub.
  2. Stage your changes using `git add` command if using Git locally.
  3. Commit the changes with a descriptive message using `git commit -m "Your message"`.
  4. Push the changes to GitHub with `git push`.

  • Ensure the page is accessible at your GitHub Pages URL: https://yourusername.github.io/webrepository

Week 7 - Topic 2

IP Address

Definition

An IP address (Internet Protocol address) is a unique string of numbers separated by periods or colons that identifies each computer using the Internet Protocol to communicate over a network.

IPv4 Address

IPv4 is the fourth version of the Internet Protocol. It uses a 32-bit address scheme allowing for 2^32 addresses. Examples:

  • 192.168.0.1
  • 10.0.0.1
  • 172.16.254.1
  • 192.168.1.1
  • 203.0.113.0

IPv6 Address

IPv6 is the sixth version of the Internet Protocol, designed to replace IPv4. It uses a 128-bit address scheme allowing for a vastly larger address space. Example:

2001:0db8:85a3:0000:0000:8a2e:0370:7334

Domain Name System (DNS)

Definition

The Domain Name System (DNS) is a hierarchical system that translates human-readable domain names into IP addresses needed by computers to communicate with each other over the Internet.

Components

  • Domain Names: The address that users type into a browser (e.g., www.example.com).
  • DNS Records: Information stored in DNS servers about domain names (e.g., A records, MX records).
  • DNS Servers: Servers that store DNS records and respond to DNS queries.

How It Works

When you enter a domain name into your browser, the DNS system translates this name into the IP address of the server hosting the website. This process involves several steps, including querying DNS servers to find the correct IP address and returning it to your browser to establish a connection.

Practical Session: Using nslookup Command

To query DNS records using the `nslookup` command, follow these steps:

  1. Open the Command Prompt (CMD) on your computer.
  2. Type the `nslookup` command followed by the domain name you want to query.
  3. Review the returned DNS records for the specified domain.

Week 8 - Topic 2

HTTP and HTTPS

Definition

HTTP (Hypertext Transfer Protocol) is the protocol used for transferring web pages on the Internet. HTTPS (HTTP Secure) is the secure version of HTTP that encrypts data for secure communication.

HTTP Methods

  • GET: Requests data from a specified resource.
  • POST: Submits data to be processed to a specified resource.
  • PUT: Updates a current resource with new data.
  • DELETE: Deletes the specified resource.

Practical Session: Exploring HTTP and HTTPS

To explore HTTP and HTTPS methods, follow these steps:

  1. Open the Developer Tools in your browser (usually F12 or right-click and select "Inspect").
  2. Navigate to the "Network" tab to view the HTTP requests made by the webpage.
  3. Inspect different network requests to see their method (GET, POST, etc.) and other details.
  4. Check the URL bar to see whether the website is using HTTP or HTTPS.

HTTP Status Code Overview

Response Codes

HTTP status codes are issued by a server in response to a client's request made to the server. These codes represent the result of the server's attempt to process the request. Here‚Äôs a brief overview of the most common HTTP status codes:

  • 100-199 Informational:
    • 100 Continue
    • 101 Switching Protocols
  • 200-299 Success:
    • 200 OK
    • 201 Created
    • 202 Accepted
    • 204 No Content
  • 300-399 Redirection:
    • 301 Moved Permanently
    • 302 Found
    • 304 Not Modified
  • 400-499 Client Error:
    • 400 Bad Request
    • 401 Unauthorized
    • 403 Forbidden
    • 404 Not Found
  • 500-599 Server Error:
    • 500 Internal Server Error
    • 502 Bad Gateway
    • 503 Service Unavailable
    • 504 Gateway Timeout
